Pengertian Object Diagram

Object diagram merupakan sebuah gambaran tentang objek-objek dalam sebuah system pada satu titik waktu. Karena lebih menonjolkan perintah-perintah dari pada class, object diagram lebih sering disebut sebagai sebuah diagram perintah. Object diagram sangat mirip dengan diagram kelas. Perbedaan utama adalah bahwa diagram objek menggambarkan objek dan hubungan mereka. Tujuan utama dari diagram objek adalah untuk memungkinkan analis untuk mengungkap rincian tambahan kelas. Dalam beberapa kasus, pernyataan variabel dari sebuah class diagram dapat membantu pengguna atau analis dalam menemukan atribut tambahan yang relevan, hubungan, dan atau operasi, atau mungkin menemukan bahwa beberapa atribut, hubungan, atau operasi yang salah tempat. Objek diagram menggambarkan instansiasi objek-objek pada satu titik waktu tertentu dalam sistem, sehingga representasi objek-objek tersebut bersifat tetap pada saat diagram dibuat. Ini berbeda dengan diagram dinamis seperti diagram urutan atau diagram aktivitas yang menunjukkan aliran waktu atau perubahan dalam sistem. Objek diagram menampilkan objek-objek dan hubungan statis di antara mereka pada saat diagram tersebut dibuat, sehingga sifatnya statis. Diagram objek pada dasarnya adalah instance atau representasi konkret dari semua atau sebagian dari diagram kelas yang ada. Ini menggambarkan bagaimana objek-objek tersebut diinstansiasi dan berinteraksi satu sama lain pada titik waktu tertentu dalam sistem.

Fungsi Object Diagram

Berikut adalah beberapa fungsi object diagram

01

Menggambarkan seperti apa hubungan antara objek-objek yang ada

02

Melakukan pengujian terhadap Class Diagram dengan memanfaatkan objek diagram tertentu

03

Object Diagram berfungsi untuk memeriksa literasi pada sebuah program

04

Menghasilkan gambar dengan resolusi tinggi dari sebuah sistem yang sedang dikembangkan

05

Diagram objek juga digunakan untuk membuat sebuah prototype

06

Berfungsi juga dalam membuat permodelan data yang bersifat kompleks

07

Untuk memahami sebuah sistem melalui perspektif yang lebih praktis
